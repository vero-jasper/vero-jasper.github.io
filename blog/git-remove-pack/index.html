解决git pack文件过大的问题

June 8, 2020 · 1 min read

Vero

解决git中的pack文件过大的问题#

背景#

今天接手了一个项目，在clone的时候发现有146M，第一反应是.gitignore中没有将node_modules写入。 clone完查看.gitignore后发现我错了，再仔细查看后，找到原因：.git/objects/pack中的pack文件过大。

尝试解决问题#

  1. 查看pack中最大的三个文件

    git verify-pack -v .git/objects/pack/{pack名}.idx | sort -k 3 -n | tail -3 git verify-pack -v .git/objects/pack/pack-1e730055ae389609f150b63c7b6e1821bf3ed6ef.idx

    b44d9e81b8a617cdc0aceb6cd138490f543b23db blob 574946 129817 10920
    d9b52ba8e32e7c342e6458dc71cd102a12538846 blob 79130210 71157504 75551109
    e15d067755f42ef5037bce042e1c6b6c3cdc1471 blob 81564206 74970605 564257

    2.查询大文件的文件名 选了个最大的，发现是某次提交有人将node_modules打包成zip提交了

    git rev-list --objects --all | grep d9b52ba8e32e7c342e6458dc71cd102a12538846

    d9b52ba8e32e7c342e6458dc71cd102a12538846 node_modules.zip

3.将该文件从历史记录所有的tree中删除

git filter-branch --force --index-filter 'git rm -rf --cached --ignore-unmatch node_modules.zip'

Rewrite 5028ea02fdc9b8e7f29b9c0240933044349f0dbc (31/34) (2 seconds passed, remaining 0 predicted)
WARNING: Ref 'refs/heads/dev' is unchanged

git filter-branch --force --index-filter 'git rm -rf --cached --ignore-unmatch ./node_modules.zip' --prune-empty --tag-name-filter cat -- --all

git for-each-ref --format='delete %(refname)' refs/original | git update-ref --stdin
git reflog expire --expire=now --all
git gc --prune=now
git push origin --force --all
