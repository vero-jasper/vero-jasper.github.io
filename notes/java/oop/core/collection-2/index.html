集合类库（二）#

泛型机制#

  • 只在编译时有效，在运行环境下不区分类型

底层原理#

  • 本质是参数化类型，让数据类型作为参数传递，由于实际参数可以传递各种各样广泛的数据类型，因此得名泛型

自定义泛型接口#

  • 泛型类和普通类区别：在类名后面添加了类型参数列表，可以有多个
public Class Person<T> {
private String name;
private int age;
private T gender;
...
public T getGender() {
...
}
}

自定义泛型类#

  • 实例化泛型类时应指定具体的数据类型，并且是引用数据类型而不是基本数据类型
  • 父类有泛型，子类可以选择保留或选择指定泛型列席
  • 子类除了指定或保留父类的泛型，还可以增加自己的泛型

自定义泛型方法#

  • 输入参数时，输入一个泛型参数而不是具体参数。在调用泛型方法时，要对泛型参数进行实例化
public <T1> void printArray(T1[] list) {
for (T1 t: list) {
System.out.println("t === " + t);
}
}
  • 格式
[访问权限] <泛型> 返回值类型 方法名([泛型标识 参数名称]) { 方法体; }

在继承上的体现#

  • 若B是A的一个子类或子接口，而G是具有泛型声明的类或接口，则 G<B> 并不是 G<A> 的子类型

通配符的使用#

  • 使用通配符作为泛型类型的公共父类
List<?> list = new LinkedList<>(); // 不支持元素添加

基本概念#

  • java.util.Map<K, V> 集合中存取元素的基本单位：单对元素
  • key 不能重复，且一个 key 只能对应一个 value
  • 主要实现类有：HashMap, TreeMap, LinkedHasMap, Hashtable, Properties
  • 其中 HashMap 底层采用哈希表进行数据管理
  • TreeMap 底层采用红黑树进行数据管理
  • LinkedHashMap 与 HashMap 不同之处在于内部维护了一个双向链表，链表中记录了元素的迭代顺序，也就是元素插入集合中的先后顺序
  • Map集合是面向查询优化的数据结构，在大数据量情况下查询性能优良
  • 经常用于根据key 检索 value的业务场景

常用方法#

  • V put(K key, V value) 将 key-value对 存入Map中
  • V get(Object key) 返回参数key所对应的value
  • V remove(Object key) 删除参数key对应的键值对，返回删除的value
  • Set<K> keySet() 返回此映射中包含的键的Set视图
  • Collection<K> values() 返回此映射中包含的value的Collection视图
  • entrySet()

元素放入HashMap的原理#

  • 使用元素的key调用hashCode方法获取对应的哈希值码，再由某种哈希算法计算在数组中的索引位置。
  • 若该位置没有元素，则将键值对直接放入
  • 若该位置有元素，则使用key与已有元素比较哈希值，若哈希值不同，则将该元素直接放入
  • 若key与已有元素的哈希值相同，则使用key调用equals方法与已有元素一次比较
  • 若相等则将对应的value修改，否则键值对直接放入即可

相关常量#

  • DEFAULT_INITIAL_CAPACITY HashMap的默认容量：16
  • DEFAULT_lOAD_FACTOR 默认的加载因子
  • threshold 扩容的临界值，容量加载因子 = 160.75 = 12
  • TREEIFY_THRESHOLD 若Bucket中链表长度大于该默认值则转化为红黑树存储，该数值为8
  • MIN_TREEIFY_CAPACITY 桶中的Node被树化时最小的hash表容量，64

Collections类#

  • static max 获取集合中的最大值
  • static min 集合中的最小值
  • static reverse 反转
  • static shuffle 随机置换
  • staic swap(List<?> list, int index1, int index2) 交换顺序
  • staic sort 排序
  • staic copy(target, src) 将src拷贝到target中
