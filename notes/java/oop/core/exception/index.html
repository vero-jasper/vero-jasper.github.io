异常机制（重点）#

基本概念#

  • 在java语言中指程序运行时发生的异常情况
  • java.lang.Throwable类是java语言中错误Error和异常Exception的超类
  • 其中Error类主要用于JVM无法解决的严重错误
  • Exception类主要描述因编程错误或偶然的外在因素导致的轻微错误

异常的分类#

  • java.lang.Exception类是所有异常的超类
    • RuntimeException运行时异常，也叫非检测性异常
    • IOException和其它异常，也叫检测性异常（编译阶段能被编译器检测到的异常）
  • 其中RuntimeException类的主要子类为：
    • ArithmeticException 算术异常
    • ArrayIndexOutOfBoundsException 数组下标越界异常
    • NullPointerException 空指针异常
    • ClassCastException 类型转换异常
    • NumberFormatException 数字格式异常
graph LR Throwable([Throwable类]) --> Error([Error类]) Throwable([Throwable类]) --> Exception([Exception类]) Exception --> runtime([RuntimeException异常]) runtime --> arithmetic([ArithmeticException异常]) runtime --> array([ArrayIndexOutOfBoundsException]) runtime --> null([NullPointerException]) runtime --> classCast([ClassCastException]) runtime --> number([NumberFormatException]) Exception --> io([IOException异常])

异常的避免#

  • 开发中尽量使用if条件判断来避免异常的发生
  • 但过多的if条件判断会导致代码臃肿，可读性差

异常的捕获#

  • 语法格式
try {
...
} catch (异常类型1 e1) {
...
} catch (异常类型2 e2) {
...
}
...
finally {
...
}
  • 注意事项
    • 当需要编写多个catch时，小类型应在大类型之前
    • catch(Exception e) {}懒人写法
    • finally通常用于进行善后处理

异常的抛出#

基本概念#

  • 在某些特殊情况下不能处理或者不便于处理时，就可以将该异常转移给调用者，这种方法叫做异常的抛出。 当方法执行时出现异常，则底层生成一个异常类对象抛出，此时异常代码后续的代码不再执行

语法格式#

public void show() throws IOException {
...
}

方法重写原则#

  • 方法名、参数列表、返回值类型相同，从jdk1.5开始支持返回子类类型
  • 要求方法的访问权限不能变小，可以相同或变大
  • 要求方法不能抛出更大的异常

注意#

子类重写的方法不能抛出更大的异常，不能抛出平级不一样的异常，但可以抛出一样的异常、更小的异常及不抛出异常

经验分享#

  • 若父类中被重写的方法没有抛出异常时，则子类中重写的方法只能进行异常的捕获处理
  • 若一个方法内部又以递进的方式分别调用了好几个其它方法，则建议这些方法内可以使用抛出的方法处理到最后一层进行捕获方式处理

自定义异常#

基本概念#

针对某些特定场景的异常，需要用自定义异常加以描述

实现流程#

  • 自定义xxxException异常类继承Exception类或其子类

异常的产生#

throw new 异常类型(实参)
