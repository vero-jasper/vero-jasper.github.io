IO流#

基本概念#

  • IO是输入和输出的含义
  • IO流指读写数据时像流水一样从一端流到另一端

基本分类#

  • 按照读写数据的基本单位不同，分为字节流和字符流

    • 字节是以字节为单位进行数据读写的流，可以读写任意类型文件
    • 字符流是以字符(2个字节)为单位进行数据读写的流，只能读写文本文件

  • 按照读写数据的方向不同，分为输入流和输出流（站在程序角度）

    • 输入流主要指从文件中读取的数据内容输入到程序中，即读文件
    • 其中输出流主要指将程序中的数据内容输出到文件中，即写文件

  • 按照流的角色不同分为节点流和处理流

    • 节点流主要指直接和输入输出源对接的流
    • 处理流主要指需要建立在节点流的基础之上的流

FileWriter类（重点）#

基本概念#

  • java.io.FileWriter类主要用于将文本内容写入到文本文件

常用方法#

  • FileWriter(String fileName) 根据参数指定文件名构造对象（文件不存在则创建，有则清空后写入）

  • FileWriter(String fileName, boolean append) 以追加的方式根据参数指定的文件名构造（文件不存在则创建，有责追加文件）

    每当写入一个字符后，文件中的读写位置往后移动一位

  • void write(int c) 写入单个字符

  • void write(char[] cbuf, int off, int len) 将指定字符数组中从偏移量off开始的len个字符写入此文件输出流

  • void write(char[] cbuf) 将cbuf.length个字符从指定字符数组写入此文件的输出流

  • void flush() 刷新流

  • void close() 关闭流对象并释放有关的资源

FileReader类（重点）#

基本概念#

  • java.io.FileReader类用于从文本文件中读取文本数据内容

常用方法#

  • FileReader(String fileName) 根据参数指定的文件名构造
  • int read() 读取单个字符的数据并返回，返回-1表示读取到末尾
  • int read(char[] cbuf, int offset, int length) 从输入流中将最多len个字符的数据读入一个字符数组中，返回读取到的字符个数，返回-1则读到末尾
  • int read(char[] cbuf) 从此输入流中将最多cbuf.length个字符的数据读入字符数组中，返回读取到的字符个数，返回则读到末尾

字符编码#

  1. 编码表的由来
  • 计算机智能识别二进制数据，早期就是电信号，为了方便计算机可以识别各个国家的文字，就需要将各个国家的文字 采用数字编号的方式进行描述并建立对应的关系，该表就叫做编码表
  1. 常见的编码表
  • ASCII: 美国标准信息交换吗，使用一个字节的低7位二进制进行表示
  • ISO8859-1: 拉丁码表，欧洲码表，使用一个字节的8位二进制进行表示
  • GB2312: 中国的文字编码表，最多使用两个字节16位二进制进行表示
  • GBK: 中国的中文编码表升级，融合了更多的中文文字符号，最多使用两个字节16位二进制表示
  • Unicode: 国际标准码，融合了目前人类使用的所有字符，为每个字符分配唯一的字符码，所有文字都用两个字节16位二进制来表示
  1. 编码的发展
  • 面向传输的众多UTF(UCS Transfer Format)标准出现，utf-8就是每次8个位传输数据， 而utf-16就是每次16个位，这是为传输而设计的编码并使编码无国界，这样可以显示全世界所有文化的字符
  • Unicode 只是定义了一个庞大的、全球通用的字符集，并为每个字符规定了唯一缺的的编码，具体存储成什么样的字节流， 取决于字符编码方案，推荐的Unicode编码是utf-8和utf-16
  • utf-8: 变长的编码方式，可用1~4个字节来表示一个字符

ObjectOutputStream类#

基本概念#

  • java.io.ObjectOutputStream类主要用于将一个对象的所有内容整体写入到输出流中
  • 只能将支持java.io.Serializable接口的对象写入流中
  • 类通过实现java.io.Serializable接口以启用其序列化功能
  • 所谓序列化主要指将一个对象需要存储的相关信息有效组织成字节序列的转化过程

常用方法#

  • ObjectOutputStream(OutputStream out) 根据参数指定的引用来构造
  • void writeObject(Object obj) 将参数指定的对象整体写入到输出流
  • void close()

ObjectInputStream类#

基本概念#

  • java.io.ObjectInputStream类主要用于从输入流中一次性将对象整体读取出来
  • 所谓反序列化主要指将有效组织的字节序列恢复为一个对象及相关信息的转化过程

常用方法#

  • ObjectInputStream(InputStream in)
  • Object readObject()
  • void close()

序列化版本号#

  • 序列化机制是通过在运行时判断类的serialVersionUID来验证版本一致性的。在进行反序列化时，JVM会把传来的 字节流中的serialVersionUID与本地相应实体类的UID进行比较，如果相同则认为是一致的，可以进行反序列化， 否则就会出现序列化版本不一致的异常InvalidCastException

transient关键字#

  • transient是java语言的关键字，用来表示一个域不是该对象串行化的一部分，当一个对象被串行化的时候， transient型变量的值不包括在串行化的表示中，然而非transient型的变量是被包括进去的。

RandomAccessFile类#

基本概念#

  • java.io.RandomAccessFile类主要用于对随机访问文件的读写操作

常用方法#

  • RandomAccessFile(String name, String mode) 根据参数指定的名称和模式构造对象
    • r: 只读
    • rw: 打开以便读写
    • rwd: 打开以便读写，同步文件内容的更新
    • rws: 打开以便读写，同步文件内容和元数据的更新
  • int read() 读取单个字节的数据
  • void seek(long pos) 用于设置此文件开头测量的文件指针偏移量
  • void close()
