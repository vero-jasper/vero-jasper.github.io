MySQL索引#

/*
索引概念
可以通过对数据表中的字段创建索引，来提高查询速度
常见的索引分类
主键索引(PRIMARY KEY) 主键是一个唯一性的索引，每个表中只能有一个主键
唯一索引(UNIQUE) 索引列的所有数据只能出现一次，必须唯一
普通索引(INDEX) 最常见的索引，作用就是提高对数据的访问速度
表对于的索引被保存在一个索引文件中，如果对数据进行增删改操作，那么mysql就需要对索引进行更新
*/
/*
主键索引的创建
PRIMARY KEY(id)
1.创建表时，直接添加主键
2.创建表之后，添加索引
ALTER TABLE 表名 ADD PRIMARY KEY(指定主键列)
*/
/*
唯一索引的创建
1.创建表时，直接添加主键
UNIQUE
2.创建表之后，添加索引
CREATE UNIQUE INDEX 索引名 ON 表名(列名(长度))
CREATE UNIQUE INDEX index_test ON demo01(hobby);
添加唯一索引的列，其所有值都只能出现一次
唯一索引保证了数据的唯一性，也提升了查询效率
*/
/*
普通索引
1.CREATE INDEX 索引名 on 表名(列名[长度]);
2.ALTER TABLE 表名 ADD INDEX 索引名(列名);
*/
/*
删除索引
ALTER TABLE 表名 DROP INDEX 索引名;
*/
/*
索引的总结
创建索引的原则
优先选择为经常出现在查询条件或排序分组后的字段创建索引
索引的有点
1.可以大大提高查询速度
2.减少查询中分组和排序的时间
3.通过创建唯一索引来保证数据的唯一性
索引的缺点
1.创建和维护索引需要时间，数据量越大，时间越长
2.表中的数据进行增删改操作时，索引也需要进行维护，降低了维护的速度
3.索引文件需要占据磁盘空间
*/

视图#

/*
视图
是由查询结果形成的一张虚拟表
作用
1.权限控制
2.简化复杂的多表查询
语法
CREATE VIEW 视图名[字段列表] as SELECT 查询语句;
view 表示视图，一般跟后面的查询语句相同
as 表示跟视图提供数据的查询语句
*/
-- 创建视图
-- 查询语句
-- SELECT * FROM products p LEFT JOIN category c ON p.`id` = c.`id`;
-- 创建视图
-- CREATE VIEW products_category_view AS
-- SELECT * FROM products p LEFT JOIN category c ON p.`id` = c.`id`;
-- 操作视图，相当于操作一张只读表
/*
视图和表的区别
1.视图是建立在表的基础之上的
2.通过视图，不要进行增删改操作，视图主要就是用来简化查询的
3.删除视图，表不受影响
*/

存储过程#

/*
存储过程
多条SQL预计的合并，中间加入了一些逻辑控制
存储过程的优缺点
优点
1.调试完成就可以稳定运行（在业务需求相对稳定的情况下）
2.存储过程可以减少业务系统与数据库的交互
缺点
1.互联网项目中较少使用存储过程，因为业务需求变化太快
2.移植十分困难
*/
/*
存储过程创建方式1
语法格式
delimiter $$ -- 声明语句的结束符号
CREATE procedure 存储过程名称() --声明存储过程
BEGIN -- 开始编写存储过程
-- 要执行的SQL
END $$ -- 存储过程结束
-- 调用存储过程
CALL 存储过程名称;
*/
/*
存储过程创建方式2
语法格式
CREATE PROCEDURE 存储过程名(IN 参数名 参数类型)
delimiter $$
CREATE PROCEDURE goods_proc(IN goods_id INT)
BEGIN
-- 要执行的SQL
END $$
-- 调用存储过程，传递参数
CALL goods_proc(1);
*/
/*
存储过程创建方式3
1.变量的赋值
SET @变量名=值
2.OUT 输出参数
OUT 变量名 数据类型
delimiter $$
CREATE PROCEDURE orders_proc(IN o_oid INT, IN o_gid INT, IN o_price INT, OUT out_name INT)
BEGIN
-- 执行sql
INSERT INTO orders VALUES(o_oid, o_gid, o_price);
-- 设置out_num的值为1
SET @@out_num = 1;
SELECT @out_num;
END $$
CALL orders_proc(1,2,1000,@out_num);
*/

MySQL触发器#

/*
触发器
触发器创建的四个要素
1.监视地点(table)
2.监视事件(insert/update/delete)
3.触发时间
4.触发事件(insert/update/delete)
触发器的创建
语法结果
delimiter $ -- 自定义结束符号
create trigger 触发器名称
AFTER/BEFORE(INSERT/UPDATE/DELETE) --触发时机和监视的事件
ON 表名 -- 触发器所在表
FOR EACH row -- 固定写法，表示行触发器
BEGIN
-- sql语句
END $
*/
/*
监视的表 orders
监视的事件 INSERT
触发的时间 AFTER
触发的事件 UPDATE
1.修改结束符号
delimiter $
2.创建触发器
CREATE TRIGGER t1
3.设置触发的时间、监视的表、监视的事件
AFTER INSERT ON orders
4.行触发器
FOR EACH ROW
5.触发后要执行的操作
BEGIN
-- 触发后要执行的操作
END $
*/

DCL控制语句#

/*
DCL控制语句
创建用户
语法结构
CREATE USER '用户名'@'主机名' IDENTIFIED by '密码';
主机名为% 表示任意电脑都能登录
用户授权
语法结构
GRANT 权限1,权限2 ... ON 数据库名.表名 TO '用户名'@'主机名'
GRANT SELECT ON db2.`products` TO ....... -- 查询权限
GRANT ALL ON ....... -- 所有权限
查看权限
SHOW GRANTS for '用户名'@'主机名';
删除用户
DROP USER '用户名'@'主机名'
查询用户
SELECT * FROM USER;
*/
